# Security

GhostLedger is designed to minimize sensitive data exposure.
Internal scoring logic, thresholds, and respondent intelligence are not publicly accessible.

## Reporting vulnerabilities

Please do not open public GitHub issues for security findings.

Instead, use GitHub Private Security Advisory feature and include:

- Steps to reproduce
- Expected vs actual behavior
- Impact assessment
- Relevant logs or screenshots (if available)

We will acknowledge valid reports and coordinate remediation responsibly.

## In scope

- API endpoints
- Authentication or authorization bypass
- Data exposure (PII or scoring internals)
- Injection or execution vulnerabilities

## Out of scope

- Rate limiting limitations
- Social engineering
- Issues requiring physical access
